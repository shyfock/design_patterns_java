CARACTERÍSTICAS PRINCIPALES DE LOS PATRONES DE DISEÑO MÁS CONOCIDOS

  1. Patrones creacionales
    1. Builder:
      1. Permite la instanciación de objetos de manera rápida y fluida
      2. Aplicable cuando se requiere una herramienta de creación de múltiples objetos.
      3. Un ejemplo práctico puede ser la creación de una libreta de contactos.
    2. Factory:
      1. Permite empaquetar varias clases para ser instanciadas a través de una interfaz común. Es a través de la interfaz que se realiza la instanciación por medio de la diferenciación de parámetros de entrada, de ese modo el objeto se instancia en una u otra de las clases empaquetadas.
      2. Aplicable cuando se tienen clases similares que implementan todas la misma interfaz y su funcionamiento depende del valor de un parámetro de entrada.
      3. Un ejemplo práctico puede ser un listado de libros para los cuales se presentan características diferentes de acuerdo a su idioma, luego al instanciar un objeto libro bajo el patrón factory y recibir el parámetro de idioma se dan propiedades y métodos diferentes de acuerdo al valor de idioma ingresado.
    3. Prototype:
      1. Permite realizar una copia de un objeto previamente instanciado obteniendo todas las propiedades y métodos del original. Este objeto se copia por medio de la implementación de la interfaz Cloneable y luego pueden modificarse sus propiedades y aplicar sus métodos sin afectar el objeto original.
      2. Aplicable para la creación de múltiples objetos similares que pueden tener pequeñas diferencias en sus propiedades.
      3. Un ejemplo práctico puede ser la creación de una base de datos donde los objetos son productos para los cuales se debe cambiar el precio y la zona donde se localizan. Todos los productos tienen las mismas propiedades, solo que su precio y localización debe cambiar entre uno y otro.
    4. Singleton:
      1. Permite crear una clase sobre la cual solo puede hacerse una instancia. Cualquier nuevo objeto de la clase se realiza sobre el mismo espacio de memoria, lo que garantiza que no existirá un nuevo objeto instanciado de la misma clase.
      2. Aplicable para manejo de sesiones individuales de usuario, contraseñas de acceso y demás elementos que requieran ser individuales y no copiados o reproducidos dentro del mismo sistema.
      3. Un ejemplo práctico puede ser la asignación de contraseñas en una red de navegación, donde un usuario no puede tener dos sesiones abiertas al mismo tiempo.
  2. Patrones estructurales
    1. Adapter:
      1. Permite hacer adaptaciones de código externo que no puede modificarse o está desactualizado. Adaptar nuevas librerías para utilizar código existente.
      2. Aplicable para adaptar código de terceros a nuestro código o adaptar nuestro propio código para el uso de código actualizado.
      3. Un ejemplo práctico puede ser si quieres unificar una lista de contactos que provienen de diferentes sistemas pero traen la misma información. Desde tu sistema puedes realizar las adaptaciones necesarias para que al ingresar contactos desde cada sistema de almacenamiento puedas condensar una única lista.
    2. Decorator:
      1. Permite mejorar una clase sin que se modifique la clase original. Pueden adicionarse propiedades y métodos a través de una clase “decorator”.
      2. Aplicable cuando se pretende crear objetos de la misma clase pero con propiedades mejoradas, lo cual se logra mediante envoltorios o clases envolventes.
      3. Un ejemplo práctico puede ser la implementación de una base de datos de libros en la cual se quieren adicionar clasificaciones según opiniones de usuarios, por ejemplo: “más leído”, “más me gusta”, etc. Estas clasificaciones pueden adicionarse al objeto libro sin modificar la clase Libro a través de una clase decorator.
    3. Facade:
      1. Permite trasladar o transferir la complejidad del código a una clase desde donde se pueden realizar las acciones del código de forma sencilla. De alguna forma es como esconder el código que puede ser complejo para facilitar su ejecución.
      2. Aplicable durante el proceso de remanufactura de código o también para optimización.
      3. Un ejemplo práctico puede ser la remanufactura de código llevando el código complejo a unas cuantas líneas para su ejecución.
